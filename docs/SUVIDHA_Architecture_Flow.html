3 · Authentication Architecture

Two separate auth layers — admin JWT + citizen OTP session

Admin JWT

POST /admin/login OAuth2PasswordRequestForm → bcrypt verify → JWT HS256
Token Claims sub (username), role, department, exp (8h)
Role: super_admin All departments. Kiosk config. Merchant setup.
Role: dept_admin Own department only. Auto-scoped in all queries.

Citizen OTP Session

Step 1: Start Session POST /kiosk/session/start → generate 6-digit OTP → SHA-256 hash → SMS stub
Step 2: Verify OTP POST /kiosk/session/verify-otp → max 3 attempts, 5-min window → session_token
Session Token secrets.token_urlsafe(32) — 256 bits entropy, 30-min TTL
Returning User Phone lookup → reuse Razorpay customer ID → no re-registration

Frontend Auth (Citizen)

Register UUID user, phone regex validation, 4-digit PIN, IndexedDB storage
Login Phone lookup in IndexedDB → PIN compare → 8h session in localStorage
Rehydration On mount: read koisk_session → check expiry → load user from IndexedDB
Demo Mode loginDemo() → pre-seeded user → instant dashboard access

Security Properties

Admin passwords bcrypt via passlib CryptContext
OTP storage SHA-256 hashed — never plaintext in DB or logs
Brute force OTP: 3 attempts max then session restart required
CORS / Monitoring CORS env-configured; Sentry SDK included for production

Payment Initiation Flow

1. POST /api/v1/{dept}/pay-bill
2. svc_initiate() → orderId + isMock flag
3a. isMock=true → svc_complete() → receipt
3b. isMock=false → frontend gateway SDK
4. POST /api/v1/payments/complete
5. Webhook → signature verify → update status

Supported Methods

UPI Card (Debit/Credit) Net Banking
Per-dept Razorpay keys managed by super_admin via POST /admin/kiosk-config. Keys stored server-side; last-4 hint only returned to UI.
6 · Database Schema

11 ORM models, single declarative Base, SQLite dev / PostgreSQL prod

users

  • id (PK)
  • username, full_name, email
  • phone_number, address, city
  • hashed_password, is_active
  • last_login, google_id

admins

  • id (PK)
  • username, hashed_password
  • role: super_admin | dept_admin | merchant
  • department (NULL = super_admin)
  • merchant_config (JSONB)
  • last_login

service_requests

  • id (PK), service_request_id (unique)
  • user_id (FK → users)
  • department, service_type
  • status (state machine)
  • payload (JSONB)
  • payment_id (FK → payments)
  • created_at, completed_at

payments

  • id (PK)
  • user_id, bill_id, department
  • amount, currency
  • gateway, payment_method
  • gateway_payment_id, reference_no
  • status, paid_at

kiosk_sessions

  • id (PK)
  • full_name, phone_number, email
  • otp_code (SHA-256 hashed)
  • otp_attempts (max 3)
  • session_token (urlsafe 32B)
  • razorpay_customer_id
  • is_returning_user, kiosk_id

kiosk_config

  • id (PK)
  • department (unique: elec/water/muni/global)
  • razorpay_key_id, razorpay_key_secret
  • razorpay_key_id_hint (last 4, safe to show)
  • razorpay_mode: test | live
  • is_active, settings (JSONB)
